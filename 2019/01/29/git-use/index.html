Git命令使用简介

Jan 29, 2019 | 技术

git 初始设置

  1. 设置姓名和邮箱
    • git config –global user.name “name”
    • git config –global user.email “your_email@email.com“
      执行这个命令后会生成一个 .gitconfig文件
  2. 设置颜色
    git config –global color.ui auto
  3. 查看设置信息
    cat ~/.gitconfig

使用git前期准备

Github上连接已有仓库时的认证，是通过使用了ssh的公开密钥认证方式。
  • ssh-keygen -t rsa -C “your_email@email.com“
    来创建ssh Key
  • id_rsa 文件时私有密钥
  • id_rsa.pub 文件是公有密钥
    *cat ~/.ssh/id_rsa.pub 查看公有密钥

在github上添加ssh公钥添加完后

  • ssh -T git@github.com 进行查看是否成功

在github上建立仓库reposity

git上会有一个ssh clone URL，把他克隆到本地

  1. git clone URL
  2. 提交
    git add 将文件加入暂存区
    git commit 进行提交
  3. 进行push，将github上的代码会被更新
  4. git init用于初始化仓库
  5. git status 用于查看仓库状态
  6. git add 向暂存区中添加文件
  7. git commit 保存仓库信息，-m表示只记录一行信息
  8. git log查看提交日志
  9. git log -p 显示所有文件的改动
  10. git log -p README.md 显示rede.md文件的改动
  11. git diff 查看更改前后的差别
  12. git branch 显示分支
  13. git checkout -b 创建分支
  14. git checkout 切换分支，git checkout切换到上一层分支
  15. git merge合并分支，加上–no–ff表示在历史记录上明确记录下合并分支
  16. git log –graph表示图表显示分支
